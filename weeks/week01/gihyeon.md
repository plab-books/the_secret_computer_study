# Week 01 - 기현

## 📚 학습 내용 정리

### 1. 이진수와 십육진수
- **이진수**: 0과 1만을 사용하는 수 체계
  - 컴퓨터는 전기 신호의 on/off로 동작하기 때문에 이진수 사용
  - 예: 1010 (2진수) = 10 (10진수)
  
- **십육진수**: 0-9, A-F를 사용하는 수 체계
  - 이진수를 간결하게 표현하기 위해 사용
  - 예: 0xFF = 255 (10진수) = 11111111 (2진수)

### 2. 문자 인코딩
- **ASCII**: 7비트로 128개 문자 표현
  - 영문자, 숫자, 특수문자 표현
  - 한글 등 다국어 표현 불가능
  
- **Unicode**: 전 세계 모든 문자를 표현
  - UTF-8: 가변 길이 인코딩 (1-4바이트)
  - UTF-16: 고정/가변 길이 (2 또는 4바이트)

### 3. 비트와 바이트
- **비트(bit)**: 컴퓨터의 최소 단위, 0 또는 1
- **바이트(byte)**: 8비트의 모음
- **워드(word)**: CPU가 한 번에 처리하는 데이터 크기 (32bit, 64bit 등)

## 💭 토론 질문 답변

### Q1. 왜 컴퓨터는 이진수를 사용할까요?
컴퓨터의 기본 소자인 트랜지스터가 전기 신호의 on/off 두 가지 상태만을 안정적으로 구분할 수 있기 때문입니다. 이진수는 이러한 하드웨어 특성과 완벽하게 일치합니다.

### Q2. ASCII와 Unicode의 차이점은 무엇인가요?
ASCII는 7비트로 영문자와 기본 특수문자만 표현 가능한 반면, Unicode는 전 세계 모든 문자를 표현할 수 있습니다. UTF-8 같은 Unicode 인코딩을 사용하면 다국어 웹사이트를 쉽게 만들 수 있습니다.

### Q3. 실제 프로그래밍에서 문자 인코딩 문제를 겪어본 경험이 있나요?
Python 2에서 한글 문자열 처리할 때 UnicodeDecodeError를 자주 겪었습니다. 파일을 읽을 때 명시적으로 encoding='utf-8'을 지정해야 했던 기억이 있습니다.
